# Evaluación de Conocimientos: GitHub y Markdown

## 1. Conceptos Generales

**¿Qué es un archivo Markdown y para qué sirve?**  
> Un archivo Markdown es un archivo de texto plano que usa símbolos simples para dar formato al contenido, como títulos, listas o enlaces. Sirve para crear documentos organizados y fáciles de leer, especialmente en plataformas como GitHub.

**¿Cuál es el propósito del archivo README.md en un repositorio de GitHub?**  
*El archivo README.md sirve para explicar de qué trata un proyecto, cómo se usa y proporcionar información importante para otros usuarios.*

---

## 2. Sintaxis Markdown

| Elemento | Sintaxis Markdown | Ejemplo |
| :--- | :--- | :--- |
| Título H1 | `# Texto` | # Hola |
| Título H2 | `## Texto` | ## Hola |
| Título H3 | `### Texto` | ### Hola |
| Negrita | `**Texto**` | **Hola** |
| Cursiva | `*Texto*` | *Hola* |
| Lista desordenada | `- Elemento` | - Hola |
| Lista ordenada | `1. Elemento` | 1. Hola |

## 3. Flujo de Trabajo

**¿Qué diferencia hay entre Commit y Push?**

* **Commit:** Guarda los cambios en el repositorio local.  
* **Push:** Sube esos cambios a GitHub en internet.  

**¿Para qué sirve Fetch Origin?**  
> Sirve para ver si hay cambios nuevos en el repositorio remoto sin descargarlos.
